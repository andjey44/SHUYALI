Ты — full-stack разработчик и продуктовый инженер. У меня есть MVP веб-приложения Chill для контроля продуктов на кухне. Сейчас проект состоит из frontend-файлов index.html, style.css и script.js, которые лежат в папке projects. Приложение уже умеет показывать продукты в холодильнике, сроки годности, рецепты, список покупок и аналитику, но пока работает как обычный статический сайт.

Твоя задача — перевести проект в полноценное full-stack приложение, не ломая существующий frontend.

Действуй шаг за шагом:

1. Изучи структуру проекта
- Найди frontend-файлы index.html, style.css и script.js.
- Убедись, что они лежат в папке projects.
- Не перезаписывай существующий интерфейс и логику без необходимости.

2. Добавь backend на Node.js + Express
Создай package.json в корне проекта.
Добавь зависимости:
- express
- cors
- dotenv

Добавь scripts:
- start — запуск backend/server.js
- dev — запуск backend/server.js в режиме разработки через node --watch

Создай файл backend/server.js.
В нём настрой:
- Express сервер
- PORT через process.env.PORT || 3000
- cors
- express.json()
- тестовый API endpoint GET /api/status, который возвращает JSON с сообщением, что backend работает

3. Подключи frontend из папки projects
Настрой Express так, чтобы он раздавал статические файлы именно из папки projects:
- projects/index.html
- projects/style.css
- projects/script.js

Главная страница / должна открывать projects/index.html.
Добавь fallback route для будущих frontend-роутов, но не перехватывай /api.

4. Подготовь проект к онлайн-деплою
Сделай так, чтобы проект можно было запустить на Render или другом Node.js-хостинге.
Команды должны быть:
- Build Command: npm install
- Start Command: node backend/server.js

5. Опиши внесённые изменения
Создай Markdown-файл в папке notes с названием hw5-notes.md.
В файле опиши:
- что было добавлено
- зачем нужен backend
- как frontend был подключён к Express
- как был выполнен онлайн-деплой
- какие возможности это открывает дальше

Стиль оформления сделай как отчёт по домашнему заданию:
- заголовок
- проект
- что сделано
- улучшения 1–3
- процесс работы
- выводы
- что можно добавить дальше

6. Не удаляй существующий frontend
Важно: файлы projects/index.html, projects/style.css и projects/script.js должны продолжить использоваться как основа приложения.

Формат результата:
- внеси изменения в репозиторий
- сделай коммиты
- кратко сообщи, какие файлы были созданы или изменены
