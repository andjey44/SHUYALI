Ты — full-stack разработчик и backend инженер. У меня есть MVP веб-приложения Chill для управления продуктами в холодильнике. Проект уже работает через Node.js + Express и задеплоен онлайн.

Frontend находится в папке projects и состоит из:
- index.html
- style.css
- script.js

Сейчас данные хранятся через localStorage. Мне нужно перевести проект на облачную базу данных через Supabase.

Твоя задача — подключить Supabase, создать структуру базы данных и синхронизировать frontend с облаком.

Действуй шаг за шагом:

1. Создай новую папку
Внутри папки projects создай папку:
projects/supabase/

Все файлы для Supabase добавляй туда.

2. Создай структуру базы данных
Создай SQL migration файл:
projects/supabase/001_initial_schema.sql

В нём создай таблицы:

profiles
Хранит:
- email
- имя пользователя
- avatar
- дату регистрации

products
Хранит:
- продукты в холодильнике
- категорию
- срок годности
- цену
- barcode
- статус продукта
- user_id

shopping_items
Хранит:
- список покупок
- статус покупки
- количество
- источник добавления
- user_id

product_events
Хранит:
- историю действий с продуктами
- съеденные продукты
- выброшенные продукты
- удалённые продукты

favorite_recipes
Хранит:
- избранные рецепты
- просмотры
- лайки

user_settings
Хранит:
- тему приложения
- настройки уведомлений
- предпочтения пользователя
- валюту

3. Настрой PostgreSQL структуру
Добавь:
- foreign keys
- indexes
- created_at
- updated_at
- triggers
- Row Level Security (RLS)
- policies для пользователей

Сделай автоматическое создание профиля после регистрации пользователя.

4. Подключи Supabase к frontend
Создай файлы:
- projects/supabase/supabase.js
- projects/supabase/app-sync.js
- projects/supabase/config.example.js

В supabase.js:
- подключи Supabase client
- настрой auth
- создай функции:
  - signUp
  - signIn
  - signOut
  - getProducts
  - addProduct
  - deleteProduct
  - getShoppingItems
  - addShoppingItem

5. Подключи авторизацию
В index.html:
- подключи Supabase CDN
- добавь UI для:
  - регистрации
  - входа
  - выхода
- покажи статус пользователя

6. Синхронизируй frontend
В app-sync.js:
- подключи cloud sync
- синхронизируй products
- синхронизируй shopping_items
- обновляй UI после login/logout
- оставь fallback на localStorage для гостевого режима

7. Не ломай существующий frontend
Важно:
- текущий интерфейс должен продолжать работать
- script.js должен остаться основной логикой приложения
- Supabase должен дополнять приложение, а не полностью переписывать его

8. Добавь описание изменений
Создай файл:
notes/hw6-supabase-database.md

В нём опиши:
- какие базы данных были созданы
- зачем нужна каждая таблица
- как работает cloud sync
- как работает авторизация
- как был подключён Supabase
- какие возможности появились после интеграции

Стиль оформления:
- заголовок
- проект
- что сделано
- улучшения 1–3
- процесс работы
- выводы
- что можно добавить дальше

Формат результата:
- внеси изменения напрямую в GitHub репозиторий
- сделай коммиты
- кратко перечисли созданные файлы
