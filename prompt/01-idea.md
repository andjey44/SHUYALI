Ты — продуктовый менеджер с опытом работы в сфере Fintech и бытовых сервисов.

Предложи 3 идеи для MVP мобильного приложения, решающего проблему нерационального использования продуктов питания, которые можно реализовать за учебный курс с помощью no-code / AI-инструментов (Bolt.new, Lovable, Cursor, Supabase). Цель — помочь пользователям сократить пищевые отходы, сэкономить бюджет и время на планирование меню.

Требования к идеям:

решают конкретную «боль» пользователей (забытые продукты в холодильнике, сложности с планированием меню, дублирование покупок);
имеют чётко определённую целевую аудиторию;
MVP можно собрать за 4–6 недель командой из 2–3 человек;
включают минимум: каталог (продуктов/рецептов/сроков годности), форму ввода (добавление продуктов), личный кабинет (отслеживание запасов).
Для каждой идеи укажи:

Название (краткое, цепляющее, отражающее суть MVP).
Проблему, которую решает MVP (опиши 3–5 предложениями: как сейчас пользователи справляются с этой проблемой, какие у них «боли»).
Целевую аудиторию (например, «молодые родители с ограниченным бюджетом», «студенты, живущие в общежитии», «офисные работники, готовящие обеды дома»).
Ключевые 3 функции MVP (с кратким описанием механики работы каждой функции).
Почему это реально сделать как MVP с помощью no-code / AI-инструментов (укажи, какой инструмент подойдёт для каждой функции и почему реализация займёт 4–6 недель).
