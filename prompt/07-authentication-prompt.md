Ты — full-stack разработчик и backend инженер. У меня есть веб-приложение Chill для управления продуктами на кухне. Проект уже подключён к Supabase, есть таблицы для пользователей, продуктов, покупок, событий и настроек.

Frontend находится в папке projects и состоит из:
- index.html
- style.css
- script.js

Supabase-файлы находятся в папке:
- projects/supabase/

Сейчас нужно добавить полноценную аутентификацию пользователей, профиль и разделение данных между аккаунтами.

Твоя задача — сделать так, чтобы пользователь мог зарегистрироваться, войти, выйти из аккаунта и видеть только свои продукты и покупки.

Действуй шаг за шагом:

1. Добавь UI для аутентификации
В index.html добавь блок входа/регистрации.

В блоке должны быть:
- поле email
- поле password
- кнопка Войти
- кнопка Регистрация
- кнопка Выйти после входа
- текстовый статус пользователя

Размести блок аккуратно в правом верхнем углу, чтобы он не портил внешний вид главной страницы.

2. Подключи Supabase Auth
В projects/supabase/supabase.js добавь функции:
- getSession()
- getUser()
- signUp(email, password)
- signIn(email, password)
- signOut()

Регистрация и вход должны работать через Supabase Auth.

3. Добавь профиль пользователя
Создай файл:
projects/supabase/profile-ui.js

В нём реализуй карточку профиля пользователя.

В профиле нужно показывать:
- email пользователя
- имя пользователя, если оно есть
- количество продуктов
- количество покупок
- статус облачной синхронизации

Также добавь функции:
- getProfile()
- updateProfile()

Профиль должен быть связан с таблицей profiles.

4. Добавь синхронизацию после входа
В projects/supabase/app-sync.js сделай так, чтобы после входа:
- загружались продукты пользователя
- загружался список покупок пользователя
- обновлялся интерфейс
- обновлялась карточка профиля

После выхода приложение должно возвращаться в гостевой режим.

5. Исправь регистрацию
Сделай понятные сообщения об ошибках:
- неверный email
- короткий пароль
- email уже зарегистрирован
- неподтверждённая почта
- неверный email или пароль

Если в Supabase включено подтверждение email, объясни, что его нужно отключить для MVP. После отключения подтверждения новые пользователи должны входить сразу после регистрации.

6. Раздели данные пользователей
Сделай так, чтобы каждый пользователь видел только свои данные.

Для этого:
- включи Row Level Security для таблиц
- добавь RLS policies
- используй поле user_id
- фильтруй getProducts() по текущему user_id
- фильтруй getShoppingItems() по текущему user_id
- deleteProduct() должен удалять только записи текущего пользователя

Таблицы, для которых нужно включить RLS:
- profiles
- products
- shopping_items
- product_events
- favorite_recipes
- user_settings

7. Создай SQL migration
Создай файл:
projects/supabase/002_enable_rls_user_isolation.sql

В нём опиши:
- включение RLS
- удаление старых policies, если они есть
- создание policies для select, insert, update, delete
- правило: auth.uid() должен совпадать с user_id

Для profiles используй правило:
- auth.uid() = id

8. Не ломай гостевой режим
Если пользователь не вошёл:
- приложение должно продолжать работать через localStorage
- продукты и покупки должны сохраняться локально
- интерфейс не должен ломаться

9. Добавь отчёт
Создай файл:
notes/hw7-authentication.md

В нём опиши:
- что было сделано для аутентификации
- как работает регистрация и вход
- как устроен профиль пользователя
- как исправлено подтверждение email
- как разделены данные между пользователями
- какие файлы были изменены
- выводы и дальнейшие улучшения

Стиль оформления:
- заголовок
- проект
- что сделано
- улучшения 1–4
- процесс работы
- добавленные и изменённые файлы
- выводы
- что можно добавить дальше

- сделай коммиты
