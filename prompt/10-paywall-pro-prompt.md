Мне нужно реализовать первую монетизацию MVP: аккаунты разных типов, 7-дневный пробный период и подписку Pro.

Цель:
Добавить paywall и модель подписки, чтобы новый пользователь мог 7 дней пользоваться сайтом бесплатно, а после окончания trial для продолжения работы сайта нужна была подписка Pro.

Действуй шаг за шагом.

1. Спроектируй модель монетизации
Предложи для Chill простую подписочную модель:

Гостевой режим:
- пользователь не вошёл в аккаунт;
- данные могут храниться локально через localStorage;
- сайт не должен ломаться.

Trial:
- пользователь зарегистрировался;
- получает 7 дней бесплатного доступа;
- все основные функции доступны.

Pro:
- пользователь оплатил подписку или активировал её в тестовом режиме;
- все функции сайта доступны.

Expired:
- trial закончился;
- Pro не активен;
- основные функции сайта блокируются paywall.

2. Создай таблицу subscriptions в Supabase
Создай SQL migration:
projects/supabase/003_subscriptions_paywall.sql

В таблице subscriptions должны быть поля:
- id;
- user_id;
- plan;
- status;
- trial_started_at;
- trial_ends_at;
- current_period_end;
- stripe_customer_id;
- stripe_subscription_id;
- created_at;
- updated_at.

Статусы:
- trialing;
- active;
- past_due;
- canceled;
- expired.

Логика trial:
trial_ends_at должен автоматически выставляться на 7 дней после создания подписки.

3. Настрой RLS для subscriptions
Включи Row Level Security.

Создай policies:
- пользователь может читать только свою подписку;
- пользователь может создать только свою подписку;
- в MVP-режиме пользователь может обновить свою подписку для тестовой активации Pro.

Добавь комментарий, что в production обновление статуса подписки должно происходить только через backend или Stripe webhook, а не напрямую с frontend.

4. Обнови handle_new_user()

5. Добавь методы подписки во frontend
В файле:
projects/supabase/supabase.js

Добавь методы:

getSubscription()
Получает подписку текущего пользователя.

activatePremiumTestMode()
Активирует Pro в тестовом режиме на 1 месяц. Это MVP-режим без настоящей оплаты.

Возвращает true, если:
- status = active;
- или status = trialing и trial_ends_at ещё не прошёл.

getTrialDaysLeft(subscription)
Считает, сколько дней осталось до конца trial.

10. Сохрани гостевой режим
Важно: если пользователь не вошёл в аккаунт, сайт не должен ломаться.

В гостевом режиме:
- данные остаются в localStorage;
- paywall не должен блокировать сайт;
- регистрация и вход должны оставаться доступными.

11. Объясни, почему оплата пока тестовая
В отчёте укажи, что реальная оплата не подключается напрямую на frontend, потому что нельзя хранить Stripe Secret Key в браузере.

Правильная production-схема:
Пользователь нажимает «Купить Pro»
↓
Frontend отправляет запрос на backend
↓
Backend создаёт Stripe Checkout Session
↓
Пользователь оплачивает подписку
↓
Stripe отправляет webhook
↓
Backend обновляет subscriptions в Supabase
↓
status становится active
↓
Paywall исчезает
