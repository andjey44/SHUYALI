Вот три варианта промтов для другой ИИ — с низкой, средней и высокой детализацией. Каждый из них соответствует твоему запросу: описать пользователя, проблему, 3–5 функций и клиентский путь для приложения по ведению списка продуктов, отслеживанию сроков годности и подбору рецептов.

Слабый промт (низкая детализация)
Опиши пользователя, его проблему, 3–5 функций приложения для ведения списка продуктов, отслеживания сроков годности и подбора рецептов, а также кратко опиши клиентский путь. Формат — текст, 3–5 страниц.

Средний промт (средняя детализация)
Составь подробное описание для приложения, которое помогает вести актуальный список продуктов, отслеживать сроки годности и подбирать рецепты. Включи:
- портрет типичного пользователя (возраст, образ жизни, мотивация);
- описание основной проблемы, которую решает приложение;
- 3–5 ключевых функций с кратким описанием каждой;
- пошаговый клиентский путь: от установки приложения до регулярного использования;
- примеры сценариев использования.
Формат — связный текст на 3–5 страниц.

Сильный промт (высокая детализация)
Развернуто опиши концепцию приложения для управления продуктами на кухне. Включи:
1. Портрет пользователя: опиши 2–3 типичных персонажа (например, молодая семья, студент, занятый профессионал), их привычки, боли и ожидания от приложения.
2. Проблему: подробно распиши, с какими трудностями сталкиваются пользователи при хранении продуктов, контроле сроков годности и планировании питания.
3. Функции:
   - опиши 3–5 ключевых функций (например: умный список продуктов с уведомлениями о сроках годности, автоматический подбор рецептов по остаткам, интеграция с магазинами, совместное использование списков);
   - для каждой функции приведи примеры взаимодействия пользователя с интерфейсом.
4. Клиентский путь: пошагово опиши путь пользователя — от скачивания приложения и первого запуска до формирования привычки регулярного использования. Включи эмоциональные моменты, возможные трудности и способы их преодоления.
5. Примеры сценариев: добавь 2–3 реальных примера использования приложения в повседневной жизни.
6. Формат: текст на 3–5 страниц, структурированный, с подзаголовками и логическими блоками.

Выбирай нужный вариант в зависимости от того, насколько подробный результат тебе нужен.
