Ты — фронтенд-разработчик. Создай одностраничный лендинг для сервиса управления продуктами на кухне «KitchenMind».

Стек: чистый HTML + CSS + JavaScript (без фреймворков).

Страница должна содержать:

Header — логотип, навигация (якорные ссылки: «Холодильник», «Рецепты», «Список покупок»)

Hero-секция — заголовок «KitchenMind — порядок на вашей кухне», подзаголовок «Контролируй сроки годности, готовь из того, что есть, и экономь до 25% бюджета», CTA-кнопка «Начать управлять»

Блок «Как это работает» — 3 шага с иконками:
  1. Добавь продукты (вручную или отсканируй чек)
  2. Получай напоминания о сроках годности
  3. Готовь по рецептам из остатков

Блок «Мой холодильник» — список добавленных продуктов с полями: название, категория, срок годности, статус (зелёный/жёлтый/красный). Сортировка по сроку годности (скоро испортятся — сверху). Кнопка «Добавить продукт» (открывает модальное окно с полями: название, категория (выпадающий список: овощи/фрукты, молочка, мясо/рыба, бакалея, другое), срок годности (date picker)). Кнопка «Съел» рядом с каждым продуктом — удаляет продукт и автоматически добавляет его в список покупок.

Блок «Рецепты из остатков» — кнопка «Что приготовить?», при нажатии анализирует текущие продукты из холодильника и предлагает 3 рецепта, которые можно приготовить из имеющихся. Если продукт скоро испортится (красный статус) — рецепт с ним показывается первым. У каждого рецепта кнопка «Добавить недостающее в список покупок» (если в рецепте есть ингредиенты, которых нет в холодильнике). Пример логики: если есть яйца и молоко → «Омлет», если есть картошка и мясо → «Жаркое», если есть помидоры и сыр → «Салат капрезе», и т.д. Минимум 5 простых правил.

Блок «Список покупок» — чек-лист продуктов, которые нужно купить. Продукты добавляются автоматически при нажатии «Съел» в холодильнике или из рецептов. Кнопка «Добавить вручную» (текстовое поле). Чекбокс напротив каждого продукта для отметки «Куплено» (зачёркивается и остаётся в списке или удаляется — на ваше усмотрение). Кнопка «Поделиться списком» — копирует список в буфер обмена в формате «— молоко\n— хлеб».

Блок «Статистика кухни» — карточки с данными из localStorage:
  - Сэкономлено денег (имитация: количество съеденных продуктов × 50 руб.)
  - Выброшено продуктов (количество удалённых по истечении срока)
  - Любимая категория (по количеству съеденных продуктов)

Footer — копирайт, контакты (email, telegram)

Данные хранить в localStorage:
  - kitchen_products — массив продуктов (id, название, категория, срок_годности, статус, дата_добавления)
  - shopping_list — массив продуктов для покупки
  - eaten_products — история съеденных продуктов (для статистики)

При первом запуске добавить демо-данные: 4–5 продуктов с разными сроками (один истекает завтра, один через 3 дня, остальные свежие). Список покупок пустой.

Дизайн: современный, минималистичный, «кухонная» цветовая палитра — светлый фон (#F9F9F8), акценты: мягкий зелёный (#4CAF50) для свежих продуктов, тёплый оранжевый (#FF9800) для срочных, красный (#F44336) для просроченных. Крупные кнопки (удобно на кухне). Карточки продуктов с плавными тенями. Адаптив под мобильные устройства.

Код разделить на 3 файла: index.html, style.css, script.js.
