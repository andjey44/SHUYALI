Проект: Chill — сервис управления продуктами на кухне

Что сделано

Создан одностраничный лендинг (chill/) на чистом HTML + CSS + JS с полноценной логикой управления холодильником, подбором рецептов и списком покупок. Все данные сохраняются в localStorage.

Структура файлов

chill/
 index.html   — разметка страницы (все секции, модальное окно, toast-уведомления)
 style.css    — стили (кухонная палитра, адаптив, карточки, анимации)
 script.js    — логика: продукты, рецепты, список покупок, localStorage
Интерактивные сценарии

Сценарий 1: Управление холодильником
Пользователь добавляет продукт через модальное окно: название, категория, срок годности.
Система автоматически:

Вычисляет статус свежести: 🔴 красный (до 1 дня), 🟡 жёлтый (2–5 дней), 🟢 зелёный (6+ дней)
Сортирует продукты по сроку годности — самые срочные сверху
Показывает баннер «⚠️ Скоро испортится!» с перечнем критических продуктов
При нажатии «Съел ✓» — убирает продукт из холодильника, добавляет в историю и автоматически в список покупок
Сценарий 2: Подбор рецептов из остатков
Кнопка «Что приготовить?» анализирует текущее содержимое холодильника и предлагает до 3 рецептов.
Алгоритм:

Сопоставляет названия продуктов с ключевыми словами рецептов
Рецепты с продуктами на грани просрочки показываются первыми
Для каждого рецепта показывает ✓ есть в наличии / ✕ отсутствует
Недостающие ингредиенты можно добавить в список покупок в один клик
Сценарий 3: Список покупок

Автоматически пополняется при нажатии «Съел» или из рецептов
Ручное добавление через текстовое поле (Enter или кнопка)
Чекбокс «Куплено» зачёркивает позицию, кнопка «Очистить купленное» удаляет отмеченные
«📤 Поделиться» копирует список в буфер обмена в формате ☐ молоко / ☑ яйца для отправки в мессенджер
Дополнительные функции

Фильтрация по категориям — кнопки «Все / Овощи / Молочка / Мясо / Бакалея / Другое» мгновенно фильтруют карточки
Статистика кухни — сэкономлено денег (кол-во съеденных × 50 ₽), продукты на грани просрочки, любимая категория
Демо-данные при первом запуске — 5 продуктов с разными сроками (красный / жёлтый / зелёный)
Адаптивная вёрстка — корректно отображается на мобильных устройствах
Проверка localStorage

Открыть index.html в браузере
Добавить продукт или нажать «Съел ✓»
Открыть DevTools → Application → Local Storage
Три ключа:
kitchen_products — массив продуктов в холодильнике
shopping_list — список покупок
eaten_products — история съеденных (для статистики)
После перезагрузки страницы все данные сохраняются
