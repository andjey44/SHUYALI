Домашнее задание 9 — Аналитика MVP через Microsoft Clarity
Проект: Chill — умный помощник для управления продуктами и кухней

Цель
Добавить аналитику MVP, чтобы понимать, как пользователи реально взаимодействуют с сайтом: куда кликают, где теряются, какие элементы замечают, а какие игнорируют.

Зачем нужна аналитика
Без аналитики сложно понять, насколько удобно пользователю работать с MVP.

Нужно было узнать:
- видят ли пользователи важные кнопки и CTA;
- доходят ли пользователи до регистрации и входа;
- где пользователи останавливаются или теряются;
- удобно ли пользоваться сайтом на мобильных устройствах;
- какие элементы выглядят кликабельными, но не работают;
- где появляются dead clicks и rage clicks.

Что было выбрано
Для аналитики был выбран Microsoft Clarity.

Microsoft Clarity позволяет смотреть:
- записи пользовательских сессий;
- тепловые карты кликов;
- тепловые карты скролла;
- dead clicks;
- rage clicks;
- поведение пользователей на разных устройствах.

Почему выбран Microsoft Clarity
Для MVP важно не только знать количество посещений, но и видеть реальное поведение пользователя.

Clarity помогает понять:
- заметил ли пользователь кнопку;
- нажимает ли он туда, куда ожидалось;
- не застревает ли он на форме авторизации;
- удобно ли нажимать элементы на мобильном устройстве;
- какие блоки страницы пользователи просматривают чаще всего.

Альтернативы

Вариант 1: Supabase events
Можно было создать таблицу analytics_events и вручную записывать события.

Плюсы:
- данные хранятся в Supabase;
- можно отслеживать точные события;
- можно строить SQL-запросы.

Минусы:
- нужно вручную прописывать каждое событие;
- нет записей сессий;
- нет heatmaps;
- сложнее анализировать UX.

Вариант 2: Google Analytics
Google Analytics хорошо подходит для маркетинговой аналитики.

Плюсы:
- показывает посещения;
- показывает источники трафика;
- подходит для больших сайтов.

Минусы:
- сложнее для MVP;
- меньше фокуса на UX;
- нет такого удобного просмотра поведения пользователя, как в Clarity.

Вариант 3: PostHog
PostHog — мощный инструмент продуктовой аналитики.

Плюсы:
- события;
- воронки;
- feature flags;
- session recordings.

Минусы:
- сложнее подключение и настройка;
- для учебного MVP может быть избыточным.

Вариант 4: Microsoft Clarity
Clarity был выбран как самый подходящий вариант для текущего этапа.

Плюсы:
- простое подключение;
- бесплатный инструмент;
- есть session recordings;
- есть heatmaps;
- можно быстро увидеть UX-проблемы;
- подходит для MVP.

Минусы:
- это внешний сервис;
- нужно учитывать сбор пользовательских данных;
- желательно добавить уведомление о cookies/analytics в будущем.

Что было сделано

1. Создан проект в Microsoft Clarity
Для опубликованного сайта Chill был создан проект в Microsoft Clarity.

Сайт:
https://chill-ggoj.onrender.com/

2. Получен tracking code
Clarity выдал код отслеживания с project ID:
ws22geqsz4

3. Tracking code добавлен на сайт
Код Microsoft Clarity был добавлен в файл:
projects/index.html

Код вставлен в блок head.

4. Сайт начал отправлять данные в Clarity
После деплоя Render скрипт Clarity загружается вместе с сайтом и начинает собирать аналитику поведения пользователей.

Как это работает

Пользователь открывает сайт
↓
Браузер загружает projects/index.html
↓
В head загружается Microsoft Clarity script
↓
Clarity фиксирует действия пользователя
↓
Данные появляются в Microsoft Clarity Dashboard

Что теперь можно анализировать

1. Session recordings
Можно смотреть записи пользовательских сессий и понимать, как пользователь реально двигается по сайту.

2. Heatmaps
Можно увидеть, по каким элементам пользователи кликают чаще всего.

3. Scroll maps
Можно понять, до каких блоков страницы пользователи долистывают.

4. Dead clicks
Можно найти места, где пользователи кликают, но ничего не происходит.

5. Rage clicks
Можно увидеть элементы, которые вызывают раздражение или непонимание.

6. Mobile UX
Можно проверить, удобно ли пользоваться сайтом на телефонах.

Какие вопросы можно проверить для Chill
- замечают ли пользователи кнопку «Начать управлять»;
- доходят ли до блока «Мой холодильник»;
- нажимают ли кнопку «Добавить»;
- пользуются ли регистрацией и входом;
- не мешает ли auth-блок в правом верхнем углу;
- удобно ли добавлять продукты на мобильном;
- пользуются ли пользователи рецептами;
- кликают ли по списку покупок;
- доходят ли до аналитики кухни.
