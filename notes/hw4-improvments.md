🔧 Улучшения MVP
1. Умный ввод через штрихкод + автозаполнение срока годности
Что добавить:
Интеграция со сканером штрихкодов через браузерный API (BarcodeDetector или библиотека quagga2). После сканирования — запрос к открытой базе продуктов Open Food Facts API для получения названия и категории. Срок годности пользователь вводит вручную, но поле уже предзаполнено по типичному сроку категории.

Зачем:
Главная боль — лень вводить продукты вручную. Если добавление продукта занимает 15 секунд вместо 60, конверсия в ежедневное использование вырастает в разы.

Как реализовать:

1. navigator.mediaDevices.getUserMedia({ video: true }) → видеопоток
2. BarcodeDetector.detect(videoFrame) → получить barcode
3. fetch(`https://world.openfoodfacts.org/api/v0/product/${barcode}.json`)
4. Извлечь product_name, categories_tags → заполнить форму
5. По категории подставить дефолтный срок (молочка → +7 дней, мясо → +3 дня)
2. Персонализация рецептов с учётом предпочтений и истории
Что добавить:
Система «лайков» на рецептах (сохраняется в localStorage). Алгоритм ранжирования учитывает: совпадение ингредиентов с холодильником, лайки пользователя, частоту просмотра, срочность продуктов. Фильтр по диете: «без мяса», «без глютена», «быстрее 30 минут».

Зачем:
Сейчас рецепты показываются механически. Пользователь видит «Омлет» 10 раз подряд и перестаёт пользоваться разделом. Персонализация создаёт ощущение умного помощника.

Как реализовать:

// Формула скоринга рецепта
score = matchedIngredients * 3
      + userLikes * 5
      + viewCount * 0.5
      + urgentIngredientsUsed * 10
      - dietaryPenalty * 20
Хранить в localStorage: recipe_likes, recipe_views, diet_preferences.

3. Визуальный дашборд расходов и потерь
Что добавить:
Раздел «Аналитика» с тремя графиками на библиотеке Chart.js:

Столбчатый: сколько продуктов добавлено vs съедено vs выброшено по неделям
Круговой: распределение по категориям
Линейный: динамика «сэкономленных» денег за месяц
Зачем:
Без цифр пользователь не чувствует прогресса. Увидев, что выбросил продуктов на 1 200 ₽ за месяц, он начинает относиться к приложению серьёзно — это мотивационный крючок.

Как реализовать:

1. При каждом "Съел ✓" → запись в eaten_products с датой и ценой
2. При "Удалить" с просроченным продуктом → запись в wasted_products
3. Chart.js агрегирует данные из localStorage по неделям/месяцам
4. Оценка цены: средняя цена по категории (молоко → 80 ₽, мясо → 300 ₽)

🚀 План развития
1. Улучшение ввода и онбординга (Sprint 1–2)
Интеграция BarcodeDetector API + Open Food Facts для автозаполнения по штрихкоду. Добавить обучающий тур (библиотека Shepherd.js) при первом запуске. Цель: снизить время добавления продукта с 60 до 15 секунд — это ключевой барьер для ежедневного использования.

2. Персонализация и аналитика (Sprint 3–4)
Система лайков на рецептах, фильтры по диете, дашборд потерь на Chart.js. Push-уведомления через Notification API + Service Worker: «Молоко истекает завтра — вот 3 рецепта». Без напоминаний пользователи забывают проверять приложение.

3. Переход на облачный бэкенд (Sprint 5–6)
Замена localStorage на Supabase (PostgreSQL + REST API + Auth). Это разблокирует синхронизацию между устройствами — критичная функция для семей, где несколько человек ведут один холодильник. Аутентификация через Supabase Auth (email/Google OAuth).

4. Нативное мобильное приложение (Sprint 7–9)
Перевод на React Native + Expo с сохранением бизнес-логики. Добавить сканирование камерой (expo-barcode-scanner), виджет на экран блокировки (iOS/Android) с топ-3 критических продуктов. Нативное приложение даёт доступ к камере без запроса разрешений каждый раз и работает офлайн.

5. Интеграции с внешними сервисами (Sprint 10+)

Голосовые помощники: Alexa Skill / Google Assistant Action — «Добавь молоко в холодильник»
Магазины: интеграция с API Яндекс Лавки / Самоката — кнопка «Заказать всё из списка покупок»
Умный дом: интеграция с Home Assistant через MQTT — датчик открытия холодильника автоматически напоминает проверить сроки
