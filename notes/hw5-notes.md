Домашнее задание — Перевод проекта Chill в Full-Stack приложение
Проект: Chill — умный помощник для управления продуктами и кухней

Что сделано
Используя ChatGPT + GitHub Connector, проект был переведён из обычного frontend-сайта в полноценное full-stack приложение с backend-сервером и онлайн-деплоем.

Все изменения вносились через AI и автоматически коммитились в GitHub.

Улучшение 1: Подключение backend на Node.js + Express
Что:
В проект был добавлен backend-сервер на Express. Теперь приложение работает не как набор статических файлов, а как полноценное веб-приложение с серверной частью.

Что было добавлено:
- package.json
- backend/server.js
- Express
- CORS
- dotenv
- API endpoint /api/status

Зачем:
Backend позволяет в будущем:
- подключить базу данных
- сделать авторизацию
- хранить данные пользователей
- создавать API
- подключать AI-функции
- делать синхронизацию между устройствами

Как реализовано через ChatGPT:

Промпт:
«Давай тогда выберем 2 вариант»

После этого AI:
- создал package.json
- добавил backend/server.js
- настроил Express сервер
- подключил middleware
- создал тестовый API

Улучшение 2: Перенастройка frontend на папку projects
Что:
Backend был перенастроен так, чтобы использовать существующие файлы проекта:
- projects/index.html
- projects/script.js
- projects/style.css

Теперь frontend автоматически раздаётся через Express.

Зачем:
Это объединяет frontend и backend в одну систему. Теперь сайт можно запускать как полноценное приложение через сервер.

Как реализовано через ChatGPT:

Промпт:
«перенастрой сервер, чтобы он лежал в папке projects и использовались мои файлы index.html script.js style.css»

AI:
- обновил backend/server.js
- подключил express.static()
- настроил раздачу frontend из папки projects
- добавил fallback routing
- сохранил API маршруты

Улучшение 3: Онлайн деплой приложения через Render
Что:
Проект был выложен онлайн через Render и теперь доступен по ссылке:
https://chill-ggoj.onrender.com/

Зачем:
Теперь приложение:
- доступно из интернета
- автоматически обновляется после commit в GitHub
- может использоваться как настоящий продукт
- имеет автоматический деплой

Как реализовано:

Были настроены:
- GitHub integration
- npm install
- node backend/server.js
- автоматический deploy из ветки main

Процесс работы
Шаг 1: Подключение GitHub репозитория
Был подключён GitHub репозиторий andjey44/Chill через ChatGPT Codex Connector.

Шаг 2: Проверка структуры проекта
AI проанализировал структуру проекта и нашёл frontend-файлы внутри папки projects.

Шаг 3: Создание backend
Через AI был создан Express backend и package.json.

Шаг 4: Интеграция frontend + backend
Сервер был перенастроен для работы с папкой projects.

Шаг 5: Онлайн деплой
Проект был задеплоен на Render и стал доступен онлайн.
